# Matrix
Лёгкая шаблонная библиотека для работы с матрицами на C++.

## О проекте
Пет‑проект: класс `Matrix<T>` с ручным управлением памятью. Делался, чтобы отработать RAII, правило пяти, move‑семантику и прокси‑объекты — навыки, полезные для стажировок в ядро.

## Ключевые особенности
- **RAII**: память выделяется в конструкторе, освобождается в деструкторе.
- **Правило пяти**: реализованы конструктор/оператор копирования, конструктор/оператор перемещения, деструктор.
- **Прокси‑объект `Row_matr`**: даёт синтаксис `matrix[i][j]` и проверку границ через `assert`.
- **Шаблоны + `static_assert`**: тип `T` должен быть арифметическим.
- **Порог погрешности (`threshold`)**: нужен для сравнения вещественных чисел и вычисления определителя.

## Основные операции
- Арифметика: `+`, `-`, `*` (матрица×матрица, матрица×скаляр) и составные `+=`, `-=`, `*=`.
- Транспонирование: `Transpose()` (in‑place) и `Transposing()` (новая матрица).
- Определитель: `Det()` (для неквадратных матриц возвращает `NAN`).
- Сравнение: `operator==` с учётом порога погрешности.
- Ввод/вывод: операторы `<<`, `>>`, метод `Dump()`.

## Реализация метода Гаусса (в `Det()`)

В твоём коде метод Гаусса реализован внутри `Det()` для приведения матрицы к ступенчатому виду и вычисления определителя. Это хорошая демонстрация численных методов и работы с элементарными преобразованиями строк.

Как это устроено:
- **Проверка на квадратность**: если матрица не квадратная, `Det()` сразу возвращает `NAN`.
- **Выбор ведущего элемента**: если на диагонали элемент близок к нулю (с учётом `threshold`), ищется строка ниже с ненулевым элементом в этом столбце (`FindNonZero`). Если такой строки нет — определитель равен 0.
- **Перестановка строк**: при нахождении подходящей строки она меняется местами с текущей (`SwapLines`), а знак определителя инвертируется.
- **Обнуление столбца**: для всех строк ниже текущей выполняется преобразование: `row[dest] += val * row[src]`, где `val` подбирается так, чтобы занулить элемент под диагональю.
- **Накопление результата**: определитель равен произведению диагональных элементов, умноженному на знак перестановок.

Вспомогательные методы, которые делают это возможным:
- `SwapLines(size_t lhs, size_t rhs)` — быстрая перестановка строк через обмен указателей.
- `AddLineMVal(size_t dest_ind, size_t src_ind, long double val)` — сложение строк с коэффициентом.
- `IsZero(long double val)` — проверка на «почти ноль» через порог `threshold`.

Для пет‑проекта и собеседования это сильный пункт: ты показываешь понимание численной устойчивости (выбор ведущего элемента), работу с погрешностью и низкоуровневые оптимизации (обмен указателей вместо копирования строк).

## Конструкторы
- Пустая матрица: `Matrix<int> m(2, 3);`
- Через лямбду: `Matrix<double> m(2, 2, [](int i, int j){ return i*2.0+j; });`
- Через `std::initializer_list`: `Matrix<int> m(2, 3, {1,2,3,4,5,6});`
- Через итераторы: `Matrix<int> m(2, 3, begin, end);`
- Копирование/перемещение: `Matrix<int> m2 = m1;`, `Matrix<int> m3 = std::move(m1);`
- Единичная матрица: `Matrix<double>::Identity(3);`

## Обработка ошибок
- Границы проверяются через `assert` (в режиме отладки).
- Исключений нет.
- Для неквадратной матрицы `Det()` возвращает `NAN`.

## ABI‑заметка
Класс шаблонный, реализация в заголовках: любое изменение требует перекомпиляции всех клиентов. Это хороший повод обсудить на собеседовании плюсы/минусы шаблонов vs модулей и бинарного ABI.

## Пример
```cpp
#include "matrix.hpp"
#include <iostream>

int main() {
    Matrix<double> A(2, 2, {1.0, 2.0, 3.0, 4.0});
    Matrix<double> B(2, 2, {0.0, 1.0, -1.0, 0.0});

    auto C = A + B;
    C *= 2.0;

    std::cout << C << "\n";
    std::cout << "Det = " << C.Det() << "\n";

    return 0;
}
